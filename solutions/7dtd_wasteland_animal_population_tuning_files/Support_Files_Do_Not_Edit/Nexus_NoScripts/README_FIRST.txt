REQUIREMENTS

  - 7 Days to Die using the supported 3.0-era XML structure
  - Windows 11 and Steam are the tested environment
  - Access to the game or server Mods folder
  - The game or server stopped while files are copied or removed

INSTALL

1. Extract this archive.
2. Find the folder named:

   BitWrecked_7DTD_WastelandAnimalPopulationTuning

3. Copy that whole folder into the 7 Days to Die Mods folder.

   Typical Steam location:

   C:\Program Files (x86)\Steam\steamapps\common\7 Days To Die\Mods\

4. Confirm this exact file exists after copying:

   7 Days To Die\Mods\BitWrecked_7DTD_WastelandAnimalPopulationTuning\ModInfo.xml

5. Start the game or server and check the startup log for XML errors.

MULTIPLAYER

Install this mod on the computer that controls the world: the peer host or the
dedicated server. A local copy does not change another person's server. Follow
the server owner's Easy Anti-Cheat requirements.

REMOVE

1. Stop the game or server.
2. Delete only this folder from Mods:

   BitWrecked_7DTD_WastelandAnimalPopulationTuning

3. Start the game or server and confirm normal startup.

IMPORTANT

This no-scripts edition does not provide interactive animal selection, tuning
sliders, Scan Values, Brutal Science, automatic backup, or Restore Cap. Those
features belong to the separate Windows helper package and are not required for
the XML modlet to load.

Other mods that patch the same Wasteland entity groups or biome spawn routes may
override or combine with these values. Back up saves you care about before
testing any gameplay mod.
